# 打卡记录 App 需求文档

## 一、背景

夜班工作者，每天 7:00 上班、次日 5:00 下班。上下班时按一下按钮记录实际打卡时间，方便自查迟到/早退、周期统计、导出数据。

纯本地存储，不联网，不依赖任何第三方服务。

---

## 三、业务规则

### 3.1 跨夜班次处理

一个"工作日" = 当天标准上班时间 ~ 次日标准下班时间。

以默认班次（7:00 ~ 次日 5:00）为例：

- 上班打卡 → 取日历日期作为班次日期
- 下班打卡 → 如果时间 ≤ 当天标准下班时间（5:00），归入**前一个日历日**的班次；否则归入当天班次

```
6月16日 07:02 上班打卡 → 属于 6月16日 班次
6月17日 04:58 下班打卡 → 属于 6月16日 班次（5:00 之前，归前一日）

6月17日 06:50 上班打卡 → 属于 6月17日 班次
6月18日 05:30 下班打卡 → 属于 6月18日 班次（已过 5:00，归新一天）
```

### 3.2 工资周期

用户可配置起始日（默认 1 号），统计/历史/导出都按此周期计算。

以起始日 = 15 为例：

```
2026年6月周期 = 6月15日 7:00 ~ 7月14日 5:00
2026年7月周期 = 7月15日 7:00 ~ 8月14日 5:00
```

- 查看历史时按周期翻页
- 选"6月"实际显示的是 6/15 ~ 7/14 的数据

### 3.3 迟到/早退判定

| 情况 | 判定 |
|---|---|
| 上班打卡时间 > 标准上班时间 | 迟到 |
| 上班打卡时间 ≤ 标准上班时间 | 正常 |
| 下班打卡时间 < 标准下班时间 | 早退 |
| 下班打卡时间 ≥ 标准下班时间 | 正常 |
| 有上班无下班 | 漏打下班 |
| 有下班无上班 | 漏打上班 |

### 3.4 目标与追踪

用户可设置每个周期的目标出勤天数和目标总工时。

| 指标 | 计算方式 |
|---|---|
| 当前打卡天数 | 周期内有上班打卡记录的天数 |
| 当前打卡总时长 | 周期内所有完整打卡（上班+下班）的工时之和 |
| 目标天数 | 用户设定的值（如 22 天） |
| 目标时长 | 用户设定的值（如 484 小时） |
| 距目标天数差 | 目标天数 - 当前打卡天数 |
| 距目标时长差 | 目标时长 - 当前总时长 |
| 剩余天数 | 周期结束日 - 今天（已过结束日则为 0） |
| 日均所需工时 | 距目标时长差 ÷ 剩余天数（剩余天数=0时显示"--"） |

示例（目标 22 天 / 484 小时，已打卡 8 天 / 176 小时，剩余 20 天）：

```
已打卡 8 天 / 目标 22 天    差 14 天
已累计 176.0h / 目标 484.0h  差 308.0h
剩余 20 天  →  日均还需 15.4h
```

- 超额完成（差值为负）时显示 "+X"
- 日均所需工时 > 24h 时标注"无法达成"

### 3.5 补录规则

- 补录记录标记为"补录"，与实时打卡区分
- 补录允许输入任意日期和时间
- 导出时补录记录单独标注列
- 补录不改变迟到/早退的判定逻辑（补录的 8:00 上班仍然算迟到）

### 3.6 打卡约束

- 上班已打不可重复打上班
- 下班可重复打卡，每次覆盖更新下班时间（以最后一次为准）
- 未打上班不可先打下班
- 允许修改/删除已有打卡记录

## 四、配置项

| 配置 | 默认值 | 说明 |
|---|---|---|
| 标准上班时间 | 07:00 | 迟到判定基准 |
| 标准下班时间 | 05:00（次日） | 早退判定基准，支持设置是否跨日 |
| 工资周期起始日 | 1 | 每月第几天开始算一个周期 |
| 目标出勤天数 | 22 | 每周期目标打卡天数 |
| 目标总工时 | 484.0 | 每周期目标总工时（默认 22×22） |

## 五、数据结构

### 5.1 打卡记录

```kotlin
data class ClockRecord(
    val shiftDate: String,         // 班次日期 "2026-06-16"（上班那天的日历日期）
    val clockInTime: Long?,        // 上班打卡时间戳（ms），null=未打
    val clockOutTime: Long?,       // 下班打卡时间戳（ms），null=未打
    val clockInIsManual: Boolean,  // 上班是否补录
    val clockOutIsManual: Boolean, // 下班是否补录
)
```

### 5.2 设置

```kotlin
data class AppSettings(
    val standardClockIn: LocalTime,  // 标准上班时间，默认 07:00
    val standardClockOut: LocalTime, // 标准下班时间，默认 05:00
    val isClockOutNextDay: Boolean,  // 下班是否跨日，默认 true
    val cycleStartDay: Int,          // 工资周期起始日，默认 1
    val targetDays: Int,             // 目标出勤天数，默认 22
    val targetHours: Float,          // 目标总工时，默认 484.0
)
```

### 5.3 存储

- 方式：Room（SQLite）
- 不需要网络，纯本地
- 数据库随 app 卸载清除
