# 打卡记录 App 需求文档（自用精简版）

> 面向个人使用，优先「好用、好做」，不追求上架规范与完整产品形态。  
> 完整版见 `clock-in-app-requirements.md`。

---

## 一、背景

夜班工作者，默认 **7:00 上班、次日 5:00 下班**。按按钮记录实际上下班时间，方便自查迟到/早退、看周期工时、导出备份。

**纯本地、不联网、无权限。**

---

## 二、功能范围

### 必做（MVP）

| # | 功能 | 说明 |
|---|---|---|

## 三、业务规则

### 3.1 跨夜班次（保留，不可省）

一个「班次」= 当天标准上班时间 ~ 次日标准下班时间（标准时间来自**设置**）。

默认 7:00 ~ 次日 5:00 示例：

- **上班打卡** → 班次日期 = 当天日历日期
- **下班打卡** → 若时间 ≤ 当天标准下班时间（5:00），归入**前一日历日**的班次；否则归入当天班次

```
6/16 07:02 上班 → 6/16 班次
6/17 04:58 下班 → 6/16 班次

6/17 06:50 上班 → 6/17 班次
6/18 05:30 下班 → 6/18 班次（已过 5:00）
```

修改设置中的标准上下班时间后，迟到/早退判定与跨夜归属**同步按新值计算**。

### 3.2 工资周期

**周期起始日在设置中配置**，不设固定默认为 1 号；用户自行选择每月第几天（建议可选范围 1~28，避免大小月歧义）。

**周期边界（闭区间）：**

> 一个周期 = **当月 X 日 ~ 次月 (X−1) 日**，首尾两天均计入周期。

以起始日 **X = 15** 为例：

```
2026-06 周期 = 6月15日 ~ 7月14日（含 6/15 与 7/14）
2026-07 周期 = 7月15日 ~ 8月14日（含 7/15 与 8/14）
```

以起始日 **X = 20** 为例：

```
2026-06 周期 = 6月20日 ~ 7月19日
```

**数据归属：**

- 班次按 `shiftDate`（上班那天的日历日期）归属周期
- **历史列表、周期统计、目标进度、导出** 均只展示/统计**当前所选周期**闭区间内的班次
- 切换周期时，上述页面数据一并切换

**展示：**

- 周期选择器显示明确日期范围，如 `6/15 ~ 7/14`，避免仅用「6 月」造成误解
- 默认打开**当前所在周期**

### 3.3 迟到 / 早退 / 漏打

| 情况 | 判定 |
|---|---|
| 上班时间 > 标准上班 | 迟到 |
| 上班时间 ≤ 标准上班 | 正常 |
| 下班时间 < 标准下班 | 早退 |
| 下班时间 ≥ 标准下班 | 正常 |
| 有上班无下班 | 漏打下班 |
| 有下班无上班 | 漏打上班 |

补录与实时打卡判定规则相同（8:00 补录上班仍算迟到）。

### 3.4 打卡约束

| 规则 | 说明 |
|---|---|
| 上班不可重复 | 本班次已有上班时间后，**上班按钮禁用**；想改时间走编辑/补录 |
| 先上班后下班 | 未打上班时**不可打下班**；上班后界面只提供下班操作 |
| 下班可重复 | 已上班状态下可多次打下班，**以最后一次为准** |
| 修改 / 删除 | 历史记录可编辑、删除，用于忘打、打错等纠错 |

**界面状态示意：**

```
未上班       →  [上班] 可用   [下班] 禁用
已上班未下班 →  [上班] 禁用   [下班] 可用
已下班       →  [上班] 禁用   [下班] 可用（再次点击覆盖下班时间）
```

### 3.5 补录（简化）

- 不强制 UI 上区分「补录」标签（列表里可选显示「已编辑」）
- 导出时不必单独补录列
- 允许任意日期时间输入（纠错）；归属周期仍按 `shiftDate` 判断

### 3.6 目标进度

目标值在**设置**中配置。目标进度区域展示本周期（闭区间内）的全部指标：

#### 基础指标

| 指标 | 计算方式 |
|---|---|
| 目标打卡天数 | 来自设置 `targetDays` |
| 目标总工时 | 来自设置 `targetHours` |
| 已打卡天数 | 周期内有上班记录的班次数 |
| 已累计工时 | 周期内完整打卡（上班+下班）的工时之和 |

#### 剩余（距目标还差多少）

| 指标 | 计算方式 |
|---|---|
| 剩余打卡天数 | 目标打卡天数 − 已打卡天数 |
| 剩余打卡时长 | 目标总工时 − 已累计工时 |

- 未完成：显示正数，如「还差 14 天」「还差 308.0h」
- 超额完成：差值为负，显示「+X 天」「+X.Xh」（表示已超出目标）

#### 日均指标

| 指标 | 计算方式 |
|---|---|
| 当前日均 | 已累计工时 ÷ 已打卡天数 |
| 周期剩余天数 | 从今天到周期结束日（**含今天与结束日**）的日历天数；今天已过周期结束则为 0 |
| 接下来所需日均 | 剩余打卡时长 ÷ 周期剩余天数 |

**边界：**

- 已打卡天数 = 0 时，当前日均显示 `--`
- 周期剩余天数 = 0 时，接下来所需日均显示 `--`
- 接下来所需日均 > 24h 时，标注「无法达成」

#### 展示示例

目标 22 天 / 484h，已打卡 8 天 / 176h，周期还剩 20 天：

```
目标    22 天 / 484.0h
已完成   8 天 / 176.0h
剩余    14 天 / 308.0h

当前日均           22.0h   （176 ÷ 8）
接下来所需日均      15.4h   （308 ÷ 20）
```

超额完成示例（已打 23 天 / 500h）：

```
剩余    +1 天 / +16.0h
接下来所需日均      --      （目标已达成，无需再算）
```

> 切换历史周期时，目标值仍用设置中的目标；已打卡/剩余/日均均按**所选周期**重新计算。

## 四、设置页（均可修改）

所有与时间、周期、目标相关的项集中在**设置**中，修改后立即生效（统计、目标、判定逻辑同步更新）。

| 配置项 | 说明 | 初始建议值 |
|---|---|---|
| 标准上班时间 | 迟到判定基准 | 07:00 |
| 标准下班时间 | 早退判定基准 | 05:00 |
| 下班是否跨日 | 夜班为 true | true |
| 工资周期起始日 | 每月第几天开启新周期（1~28） | **用户自设，非固定 1 号** |
| 目标出勤天数 | 每周期目标打卡天数 | 22 |
| 目标总工时 | 每周期目标工时（小时） | 484.0 |

> 首次安装可给建议默认值（如起始日 15、目标 22 天 / 484h），用户随时在设置里改。

## 五、数据结构

### 打卡记录

```kotlin
data class ClockRecord(
    val shiftDate: String,      // "2026-06-16"，归属周期的键
    val clockInTime: Long?,     // ms，null = 未打
    val clockOutTime: Long?,    // ms，null = 未打
)
```

### 设置

```kotlin
data class AppSettings(
    val standardClockIn: LocalTime,   // 标准上班时间
    val standardClockOut: LocalTime,  // 标准下班时间
    val isClockOutNextDay: Boolean,   // 下班是否跨日
    val cycleStartDay: Int,           // 工资周期起始日 1~28
    val targetDays: Int,              // 目标出勤天数
    val targetHours: Float,           // 目标总工时
)
```

### 存储

- Room 或 DataStore + 本地 JSON，二选一
- 纯本地，卸载即清；靠**导出**做备份

## 六、数据导出（必做）

导出**当前所选工资周期**闭区间内的全部班次（与历史列表范围一致）。

一种格式即可（CSV 或 JSON 二选一，推荐 CSV）：

**CSV 示例**

```csv
日期,上班时间,下班时间,工时(小时),备注
2026-06-16,07:02,04:58,21.93,迟到
2026-06-15,06:58,05:02,22.07,
2026-06-14,07:10,04:30,21.33,迟到;早退
```

文件命名建议含周期范围，如 `打卡_2026-06-15_2026-07-14.csv`。

**JSON（若选用）**：`records` 数组 + `summary`，summary 至少包含：

```json
{
  "cycleStart": "2026-06-15",
  "cycleEnd": "2026-07-14",
  "targetDays": 22,
  "targetHours": 484.0,
  "clockedDays": 8,
  "totalHours": 176.0,
  "remainingDays": 14,
  "remainingHours": 308.0,
  "currentDailyAvg": 22.0,
  "cycleDaysRemaining": 20,
  "requiredDailyAvg": 15.4
}
```

## 九、与完整版差异摘要

| 项目 | 完整版 | 自用精简版 |
|---|---|---|
| 打卡约束 | 上班不可重复、先上班后下班、下班可覆盖 | **相同** |
| 设置 | 班次 / 周期 / 目标均可配 | **相同（均必做）** |
| 工资周期 | 可配起始日，闭区间 | **相同；默认非 1 号，用户自设** |
| 目标进度 | 含日均推算等 | **相同**（目标/剩余/当前日均/接下来所需日均） |
| 补录标记 | 强制区分 + 导出列 | 可不区分 |
| 导出 | CSV + JSON | **必做，一种格式即可** |
| 周期报告卡片 | 必做 | 不做 |
| 非功能 | API26、5MB、SDK34 | 不卡 |
